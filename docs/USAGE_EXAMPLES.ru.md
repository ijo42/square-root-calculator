## Начало работы

### Первый запуск

При первом запуске приложения:
1. Интерфейс открывается на языке вашей системы (если поддерживается)
2. Тема устанавливается в соответствии с системными настройками (светлая/тёмная)
3. Настройки сохраняются в `~/.square_root_calculator/`
4. Автоматическая проверка обновлений запускается в фоновом режиме

### Базовый интерфейс

Калькулятор имеет два основных режима, доступных через вкладки:
- **Действительные числа**: Для вычисления квадратных корней положительных действительных чисел
- **Комплексные числа**: Для вычисления квадратных корней комплексных чисел (включая отрицательные действительные)

## Примеры с действительными числами

### Пример 1: Полные квадраты
**Ввод:** 16  
**Точность:** 10  
**Результат:**
- Положительный корень: `+√(16) = 4`
- Отрицательный корень: `-√(16) = -4`

Полные квадраты дают точные целочисленные результаты.

### Пример 2: Иррациональные числа
**Ввод:** 2  
**Точность:** 50  
**Результат:**
- Положительный корень: `+√(2) = 1.41421356237309504880168872420969807856967187537694...`
- Научное: `1.414213562373095e+00`
- Дробь: `1414/1000` (приближение)

Калькулятор отображает несколько представлений для лучшего понимания.

### Пример 3: Большие числа
**Ввод:** 123456789  
**Точность:** 20  
**Результат:** `11111.11106055555544054166...`

Даже очень большие числа могут быть вычислены точно с произвольной точностью.

### Пример 4: Десятичный ввод
**Ввод:** 0.25  
**Точность:** 10  
**Результат:** `0.5`

Десятичные числа обрабатываются без проблем.

### Пример 5: Очень малые числа
**Ввод:** 0.000001  
**Точность:** 15  
**Результат:** `0.001`

Калькулятор работает с числами любого масштаба.

## Примеры с комплексными числами

### Пример 1: Простое комплексное число
**Действительная часть:** 3  
**Мнимая часть:** 4  
**Точность:** 10  
**Результат:**
- Положительный корень: `2+1i`
- Отрицательный корень: `-2-1i`

Это классический пример: √(3+4i) = 2+i

### Пример 2: Чисто мнимое число
**Действительная часть:** 0  
**Мнимая часть:** 4  
**Точность:** 10  
**Результат:**
- Положительный корень: `1.414213562+1.414213562i`
- Отрицательный корень: `-1.414213562-1.414213562i`

Квадратный корень из чисто мнимых чисел.

### Пример 3: Отрицательное действительное (√-1)
**Действительная часть:** -1  
**Мнимая часть:** 0  
**Точность:** 10  
**Результат:**
- Положительный корень: `0+1i`
- Отрицательный корень: `0-1i`

Это демонстрирует √(-1) = i, фундаментальный результат в комплексном анализе.

### Пример 4: Большие комплексные числа
**Действительная часть:** 100  
**Мнимая часть:** 100  
**Точность:** 15  
**Результат:**
- Положительный корень: `11.09868411346781+4.50665869232206i`

Калькулятор легко справляется с большими комплексными числами.

## Управление точностью

### Использование ползунка
- Перетащите ползунок для регулировки точности от 1 до 200 знаков после запятой
- Идеально подходит для быстрой настройки во время исследования
- Визуальная обратная связь показывает текущее значение точности

### Использование поля ввода
- Введите точное значение от 1 до 1000 знаков после запятой
- Полезно, когда вам нужна конкретная точность
- Нажмите Enter или кнопку Вычислить для применения

### Предупреждение о низкой точности
Если точность слишком низкая для вычисления, вы увидите сообщение об ошибке:
- **Русский**: "Точность слишком низкая для данного вычисления. Пожалуйста, увеличьте точность минимум до X знаков после запятой."
- **English**: "Precision too low for this calculation. Please increase precision to at least X decimal places."

Пример, вызывающий низкую точность:
- Ввод: 13+i с точностью 1
- Решение: Увеличьте точность минимум до 3

## Пользовательские переводы

### Добавление нового языка

1. Создайте папку `translations` в директории приложения
2. Создайте JSON файл с именем `{код_языка}.json` (например, `de.json`)
3. Добавьте переводы со специальным полем `_language_name`:

```json
{
  "_language_name": "Deutsch",
  "app_title": "Quadratwurzel-Rechner",
  "calculate_button": "Berechnen",
  "clear_button": "Löschen",
  "result_label": "Ergebnis"
}
```

4. Перейдите в **Язык → Обновить переводы**
5. Ваш новый язык появится в меню языков!

### Переопределение существующих переводов

Вы можете настроить английский или русский, создав `en.json` или `ru.json` с только теми ключами, которые хотите изменить.

См. [translations/README.md](../translations/README.md) для полной документации.

## Советы и хитрости

### История вычислений
- Все вычисления автоматически сохраняются на панели истории
- Прокручивайте прошлые вычисления
- Используйте **Очистить историю** для начала с чистого листа

### Выбор темы
- Перейдите в **Настройки → Тема** для выбора светлой или тёмной темы
- Предпочтение темы сохраняется и сохраняется между сеансами
- Системная тема определяется автоматически при первом запуске

### Управление обновлениями
- Автоматическая проверка обновлений происходит при запуске
- Ручная проверка: **Справка → Проверить обновления**
- Нажмите **Скачать**, чтобы получить последнюю версию
- Нажмите **Пропустить**, чтобы закрыть уведомление (не будет показываться снова для этой версии)
- Пропущенные версии сохраняются в настройках

### Сочетания клавиш
- Нажмите Enter в полях ввода для вычисления
- Tab между полями для эффективного ввода данных

### Восстановление после ошибок
Если вы столкнулись с ошибкой:
1. Проверьте формат ввода (только числа, никакого текста)
2. Попробуйте увеличить точность
3. Для комплексного режима вводите действительную и мнимую части отдельно
4. Используйте историю для просмотра того, что работало раньше

## Распространённые сценарии использования

### Научные вычисления
Установите точность 50+ для научной работы, требующей высокой точности.

### Образовательные цели
Используйте более низкую точность (10-20) для демонстрации концепций без излишних подробностей.

### Быстрые оценки
Используйте ползунок на низкой точности (1-5) для быстрых приближений.

### Изучение комплексных чисел
Переключитесь в режим комплексных чисел для визуализации и понимания комплексных квадратных корней.
